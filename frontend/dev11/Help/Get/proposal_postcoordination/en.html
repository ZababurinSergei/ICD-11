    The Linearizations

    A linearization is a subset of the foundation component, that is:
    1. Fit for a particular purpose: reporting mortality, morbidity, or other uses
    2. Composed of entities that are Mutually Exclusive of each other
    3. Each entity is given a single parent

    Linearizations are similar to the classical print versions of ICD Tabular List (e.g. volume I of ICD-10 or other previous editions). Various linearizations could be built at different granularity, use case or other purposes such as for Primary Care, Clinical Care or Research. The linkage from the foundation component to a particular linearization will ensure consistent use of the classifications.

Postcoordination Proposals

With this option, you could propose new postcoordination rules on entities or suggest changes in their value sets

  • To allow postcoordination for an axis:
    1. Create a postcoordination proposal on the entity which you'd like to add new postcoordination rule
    2. Change the requested column to 'allowed'. If you'd like this postcoordination to be mandatory choose 'required'.
    3. Steps 1 & 2 are sufficient for most cases. However, for some axes you may limit the value set for the postcoordination. For example, if you'd like the postcoordination on specific anatomy, you may limit the postcoordination to a part of the anatomy hierarchy only.

      In such cases you'll see the icon . Clicking on this icon shows you a selection of possible values that you could check in or out .

  • Suggesting removal of a proposal is similar. You need to set the requested column to "not allowed"
  • You may also just suggest a change in the value set of an existing postcoordination rule. In that case you need to follow the Step 3 above.

Please note that the postcoordination proposals are specific to MMS at the moment. Therefore you need to be on an entity that exists in MMS in order to be able to suggest a postcoordination proposal
