    The Linearizations

    A linearization is a subset of the foundation component, that is:
    1. Fit for a particular purpose: reporting mortality, morbidity, or other uses
    2. Composed of entities that are Mutually Exclusive of each other
    3. Each entity is given a single parent

    Linearizations are similar to the classical print versions of ICD Tabular List (e.g. volume I of ICD-10 or other previous editions). Various linearizations could be built at different granularity, use case or other purposes such as for Primary Care, Clinical Care or Research. The linkage from the foundation component to a particular linearization will ensure consistent use of the classifications.

Proposals

The main goal of the Proposal mechanism is allowing experts to suggest changes in the classification or enhance the content in it

The "Proposals" menu is the entry point to the Proposal System

From here you could:

  • Add new proposals from the "Add Proposals / See in hierarchy" sub menu
  • See the list of proposals or search them using different criteria from the "Proposal List/Search" sub menu

To go back to the regular ICD-11 Browser you need to use the "Browse" menu

Proposal System supports the following 4 main proposals types

  • Content Enhancement Proposal

    With this proposal type you can propose changes, additions or deletions to the content of the current entity

    For example, you may propose changes in the definition of an entity, add or delete synonyms, narrower terms or any other properties.

    When you choose this option, the system shows you the current information (definition, synonyms, etc.) and your proposal is created by simply editing a copy of the existing content.

  • Propose Adding a new child

    You need to use this proposal type if you'd like to propose adding a new entity (e.g. a disease) to the classification. This proposal has to be added to the entity where the newly generated entity will be placed as a child.

  • Propose Deleting Current Entity

    Use this proposal type if you'd like to remove the current entity from the classification

  • Propose Complex Hierarchical Changes

    You need to use this option if you're proposing other hierarchical changes than simple addition or deletion of an entity

    For example, if you're suggesting changes in the block structure of a chapter, this is the proposal type to use.

  • Postcoordination Proposals

    With this option, you could propose new postcoordination rules on entities or suggest changes in their value sets.
